## 2026-02-28

### 本次目标
1. 上线「面试前策略质量门槛开关」
2. 保持准备策略“结果质量导向”，避免低质量证据直接输出正式行动清单

### 已完成变更
1. 数据结构扩展
   - `PrepRequest` 新增：
     - `qualityGateEnabled?: boolean`
     - `qualityGateThreshold?: number`
   - `PrepQuality` 新增：
     - `gateEnabled`
     - `gateThreshold`
     - `gatePassed`
     - `gateReason`
   - 文件：`app/src/lib/types.ts`

2. 策略引擎新增门槛逻辑
   - 门槛开关 + 阈值（默认 60，范围 40-90）
   - 门槛判定：
     - 证据得分低于阈值 => 不通过
     - 命中冲突条目 > 0 => 不通过
   - 未通过时：
     - `strategySummary` 切换为“先补证据”指引
     - `actionChecklist` 切换为“补证据动作清单”
     - `warning` 返回门槛触发原因
   - 文件：`app/src/lib/library-prep.ts`

3. API 层接入门槛参数
   - `/api/library/prep` 支持接收 `qualityGateEnabled` 与 `qualityGateThreshold`
   - 参数校验：
     - `qualityGateEnabled` 必须为布尔
     - `qualityGateThreshold` 必须在 40-90
   - 文件：`app/src/app/api/library/prep/route.ts`

4. 前端新增门槛配置与可视化
   - 表单新增：
     - 「启用质量门槛开关」
     - 「门槛阈值输入（40-90）」
   - 结果新增：
     - 门槛开关状态 / 阈值 / 是否通过
     - 未通过原因展示
   - 文件：`app/src/app/page.tsx`

5. 文档更新
   - README 新增门槛请求参数与返回字段说明
   - 文件：`app/README.md`

### 验证结果
1. `npm run lint` 通过
2. `npm run build` 通过
3. `/api/library/prep` 实测可返回：
   - `plan.quality.gateEnabled/gateThreshold/gatePassed/gateReason`

### 风险与注意
1. 当前门槛规则是 v1（评分阈值 + 冲突条目），后续可按真实使用数据调优
2. 若经验库整体质量偏低，门槛会频繁触发，需搭配“补证据流程”使用

### 下一步建议
1. 增加「门槛触发次数」埋点，衡量策略可用率
2. 提供「一键修复入口」：直接跳转到待复核/冲突条目列表
3. 支持按岗位配置不同门槛（如：AI 产品经理 65，普通产品经理 55）
